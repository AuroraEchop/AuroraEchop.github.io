拓展:工具函数和构造函数

这里区分的场景是，

  1. 工具函数：负责隐式转换类型 Number()
  2. 构造函数：负责包装对象 new Number()

所以这里 new 似乎就有一种魔力？
new  的“魔力”：触发的 4 步底层执行逻辑

当函数前加  new  时，JS 引擎会强制该函数进入构造模式，自动执行以下 4 个步骤（这是普通函数调用没有的）：
1. 创建空对象：自动创建一个全新的空普通对象（ const obj = {} ）。

2. 绑定原型链：将这个空对象的原型链指向构造函数的  prototype  属性（ obj.proto = 构造函数.prototype ）。

  • 比如  new Number()  时，空对象的  proto  会指向  Number.prototype ，因此实例能访问  toFixed()  等方法。
    ​
    3. 绑定  this  指向：将构造函数内部的  this  指向这个空对象，执行构造函数体的代码。
  • 比如  Number  构造函数会把传入的参数处理后，挂载到  this  上作为属性。
    ​
    4. 决定返回值：
  • 如果构造函数没有显式返回对象类型的值（对象、数组、函数等），则自动返回第一步创建的对象；
  • 如果构造函数显式返回了对象类型，则返回该对象（覆盖默认的新对象）
