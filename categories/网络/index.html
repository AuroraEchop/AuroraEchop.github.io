http请求
发表于2026-03-06|网络
客户端 →网关层（Nginx / Envoy / SLB / CDN） →应用层（Tomcat / Node / PHP-FPM / 业务容器） →后端服务（数据库、微服务、第三方接口） 502 是“网关自己收到无效响应”，504 是“网关没在规定时间内收到响应”。 502 Bad Gateway。一般表现为你自己写的应用层服务(Java/Go/PHP)挂了，网关层无法接收到响应 504 Gateway Timeout。一般表现为应用层服务 (upstream) 超时，如查库操作耗时十分钟，超过了 Nginx 配置的超时时间
